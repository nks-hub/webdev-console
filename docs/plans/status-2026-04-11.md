## Runtime Follow-up Closed In Iteration 5

Deep code review surfaced daemon lifecycle issues that were not reflected in the earlier status snapshot. Iteration 5 closed them:

1. `ProcessManager` restart-limit enforcement now survives flapping processes instead of resetting the counter after every restart.
2. Manual service stop no longer emits a false `crashed` event or schedules auto-restart.
3. `ApplicationStopping` now blocks on async cleanup, so service/plugin shutdown and port-file deletion complete before process exit.
4. Deleting a site now refreshes the managed hosts block, so removed domains stop resolving to `127.0.0.1`.
5. xUnit regression tests cover stop/exit lifecycle and restart-limit disabling.
6. Shutdown cleanup is now exercised through a dedicated `ShutdownCoordinator` with direct tests for service stop, plugin stop, and port-file deletion.
7. The Node e2e harness now has a `waitFor()` helper and a `Manual stop stays stopped` scenario for environment-aware lifecycle coverage.
8. An isolated lifecycle harness now starts a fresh daemon, runs the shared scenarios, requests graceful shutdown through `POST /api/admin/shutdown`, and verifies port-file cleanup end-to-end on a clean host.
